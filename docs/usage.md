# 使用说明

## 安装

- Windows x64：从 GitHub Releases 下载 `ZCode.Switcher_x.x.x_x64-setup.exe`。
- macOS Apple Silicon：下载 `ZCode.Switcher_x.x.x_aarch64.dmg`，打开后将应用拖入“应用程序”目录。

macOS 当前为未公证内测版。首次启动如被系统阻止，请打开“系统设置 → 隐私与安全”，确认允许打开 ZCode Switcher。

## 悬浮窗模式

先在设置中开启“无感切换”，再开启“悬浮窗模式”。

开启后，ZCode Switcher 会切换成椭圆胶囊窗，显示 GLM-5.2 账号池统计：

- 已用账号数量：GLM-5.2 剩余额度低于当前设置阈值的账号数。
- 剩余账号数量：总账号数减去已用账号数。
- 已用累计额度：所有账号 GLM-5.2 已用额度累计。
- 总额度：所有账号 GLM-5.2 总额度减去“账号数 × 当前阈值”，再加上已用账号低于阈值的部分。

点击胶囊右侧关闭按钮，窗口恢复普通模式。

## 隐藏账号信息

点击主界面功能栏里的眼睛按钮，可以隐藏账号列表中的邮箱和手机号区域。

再次点击该按钮，会恢复显示账号信息。

## 额度刷新

设置里的“批量刷新间隔”用于定时刷新账号列表额度；“当前账号刷新间隔”用于刷新当前登录账号。开启低额度自动切换账号时，当前账号会固定每 20 秒刷新一次，不使用手动设置的当前账号刷新间隔。

软件运行时会申请浏览器后台保活锁，尽量避免窗口最小化或放到后台后定时器被 WebView 挂起。窗口重新可见或重新获得焦点时，软件还会检查刷新任务是否已经到期；如果已经到期，会立刻补刷一次批量额度刷新或当前账号刷新，避免一直停在休眠前的额度状态。

如果电脑进入系统睡眠、网络断开，或软件进程被系统结束，软件不会强制唤醒系统继续联网刷新；这些情况要等系统恢复后再继续刷新。

## 公告

软件启动时会尝试读取公告；网络不可用或公告读取失败时，不会影响主界面打开。

点击右上角语言切换旁边的公告按钮，可以手动打开公告窗口。公告分为“系统公告”和“临时公告”两类。

公告内容来自 `public/notice.json`，发布后也会优先读取 GitHub 仓库中的同路径文件，适合放版本提醒、维护提示或短期通知。软件启动时会拉取一次远程公告，运行中每 15 分钟自动拉取一次；手动打开公告窗口时也会尝试刷新。

公告正文支持轻量标记：

- `<red>文字</red>`：显示为红色重点文字。
- `<link url="https://example.com">链接文字</link>`：显示为可点击链接。
- `<link url="https://example.com" />`：隐藏链接名，显示为“打开链接”。
- `<link url="https://example.com/file.zip" browser="false">获取</link>`：不打开浏览器，点击后先显示文件名、类型和大小供确认，再选择下载目录；软件会优先按响应头自动识别文件名和后缀，下载完成后在软件内提示。
- 直接写 `https://example.com`：会自动显示为可点击链接。

公告不会直接渲染 HTML，除上述标记外的内容会按普通文本显示。

已安装的软件不会直接读取开发目录里的 `public/notice.json`。在线时软件会优先读取 GitHub 仓库 `main/public/notice.json`，因此公告正式生效需要把该文件推送到 GitHub；安装包内置公告只在远程读取失败时兜底。更完整的发布和格式说明见 `docs/publish-and-format-guide.md`。

## 本地 API 与 API Key 供应商

本地 API 服务和 API Key 导入供应商目前临时下线。两者后续会拆成互不相关的能力：API Key 导入供应商用于写入或管理 ZCode 自定义供应商，本地 API 服务用于本机反代网关，不能再把 API Key 供应商默认挂到本地反代上游。

当前安装包不会打包验证码求解运行时和完整 `node_modules`。

## 本地代理与账号池

本地 API 服务会在本机提供统一 `Base URL` 和网关密钥。账号池用于选择已保存的 ZCode 账号作为反代上游，外部 API Key 上游作为补充来源。

### OAuth 登录添加 ZCode 账号

1. 点击功能栏导入按钮。
2. 选择“OAuth 登录添加”。
3. 软件会打开 Z.ai 授权页，请在浏览器里完成登录并授权。
4. 浏览器显示登录完成后，回到 ZCode Switcher 等待账号自动导入。

如果登录页长时间没有跳回或提示失败，通常是网络、浏览器登录状态或 Z.ai OAuth 服务变更导致，可以重新点击 OAuth 登录添加再试一次。

### 添加账号到账号池

1. 先通过 JSON/ZIP 导入或 OAuth 登录保存 ZCode 账号。
2. 在主界面找到“账号池”卡片。
3. 从下拉框选择已保存账号，点击添加按钮。
4. 加入账号池后，可以单独启用、停用或移除，不会删除原始账号档案。

### 添加外部 API Key 上游

1. 点击功能栏导入按钮。
2. 选择“API Key 服务商”。
3. 填写名称、接口地址、API Key、接口格式和模型名称。
4. 保存后，外部上游会以卡片形式显示在主界面，API Key 会加密保存在本机配置目录，不会明文显示。

接口地址建议填写到版本路径，例如：

```text
https://example.com/v1
```

如果服务商是 Anthropic 兼容接口，本地代理会转发到：

```text
https://example.com/v1/messages
```

### 启动本地代理

1. 在主界面找到“本地 API 服务”卡片。
2. 点击卡片底部的启动按钮。
3. 复制卡片中的 `Base URL` 和“密钥”到需要调用模型的工具中。

默认本地代理地址是：

```text
http://127.0.0.1:17860/v1
```

如果手动添加到 ZCode 的自定义供应商，`Base URL` 建议填写根地址：

```text
http://127.0.0.1:17860
```

ZCode 会按 Anthropic Messages 格式自行拼接 `/v1/messages`。软件启动本地 API 后会自动写入这个根地址；如果旧配置里已经填了 `/v1`，本地代理也会兼容 `/v1/v1/messages`，避免测试连接直接 404。

调用时需要带本地网关密钥，支持两种写法：

```text
Authorization: Bearer <网关密钥>
```

或：

```text
x-api-key: <网关密钥>
```

当前已支持的接口：

- `GET /`
- `GET /v1`
- `GET /v1/health`
- `GET /v1/models`
- `POST /v1/messages`
- `POST /v1/messages/count_tokens`

注意：代理只监听 `127.0.0.1`，默认只能本机访问。

如果账号池中的 ZCode OAuth 账号触发上游验证码校验，本地代理会先尝试通过阿里云无痕验证生成 `X-Aliyun-Captcha-Verify-Param`，再把验证参数带给 ZCode 上游。当前实现依赖本机可执行 `node`，安装包会随带 `scripts/captcha-runtime/` 里的完整求解运行环境，开发目录则可回退到 `scripts/captcha-solver.cjs`；如果自动验证失败，本地代理会跳过该账号继续尝试其他上游。没有可用上游时，会返回“账号池触发验证码校验”的明确错误，此时可以稍后重试，或添加外部 API Key 上游作为兜底。
